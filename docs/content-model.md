# Contentful Content Model

## Content Types

### `release` ‚Äî Music Album/EP/Single

| Field | Type | Notes |
|-------|------|-------|
| `name` | string | Album title; slugified to URL via `createSlug()` |
| `date` | date | Release date |
| `cover` | Asset | Album art image |
| `spotify` | string | Spotify album URL |
| `tracks` | Reference[] | Array of `track` entries |

### `track` ‚Äî Individual Song

| Field | Type | Notes |
|-------|------|-------|
| `pos` | number | Track position/number |
| `name` | string | Track title; slugified to URL via `createSlug()` |
| `embed` | Reference | `bandcampEmbed` entry |
| `lyrics` | string (optional) | Plain-text lyrics |
| `spotify` | string (optional) | Spotify track URL |

### `bandcampEmbed` ‚Äî Bandcamp Track Embed Config

| Field | Type | Notes |
|-------|------|-------|
| `trackId` | string | Bandcamp track ID for the embed iframe |
| `name` | string | Track name |
| `album` | Reference (optional) | `bandcampAlbum` entry |

### `bandcampAlbum` ‚Äî Bandcamp Album Reference

| Field | Type | Notes |
|-------|------|-------|
| `id` | number | Bandcamp album ID |
| `url` | string | Bandcamp album URL |

### `art` ‚Äî Art Piece

| Field | Type | Notes |
|-------|------|-------|
| `title` | string | Art title; slugified via `urlize()` for URL |
| `date` | date | Creation date |
| `media` | string (optional) | Medium (e.g., "oil on canvas") |
| `lowRez` | Asset | Low-resolution image (used in gallery listing) |
| `hiRez` | Asset | High-resolution image (used on detail page) |
| `ratio` | string | Aspect ratio hint for layout (e.g., "4:3") |

### `blahg` ‚Äî Blog Post

| Field | Type | Notes |
|-------|------|-------|
| `url` | string | **Author-controlled slug** used directly as URL path segment |
| `createdAt` | date | Publication date |
| `title` | string | Post title |
| `content` | string (optional) | Markdown body; converted to HTML via Showdown |
| `heroImage` | Asset (optional) | Header image for the post |

## TypeScript Types

### Auto-Generated (`src/types/contentful.d.ts`)

Generated by `contentful-typescript-codegen` from the live CMS schema. **Regenerate after any CMS schema change:**

```bash
pnpm generate-types
```

Contains the `TypeSkeleton` interfaces used when calling the Contentful SDK with `getEntries<Skeleton>()`.

### Hand-Written (`src/types/contentful-response.ts`)

Flat interfaces that mirror the raw API response shape after unwrapping Contentful's envelope. These must be kept in sync manually if CMS field names change:

- `ContentfulAsset` ‚Äî `{ description, file: { url } }`
- `ContentfulRelease`, `ContentfulTrack`, `ContentfulBandcampEmbed`, `ContentfulBandcampAlbum`
- `ContentfulArt`, `ContentfulBlahg`
- `ContentfulResponse<T>` ‚Äî generic wrapper `{ items: T[] }`

## Service Layer

**`src/services/contentful-service.ts`** ‚Äî singleton `ContentfulService` class with one method per content type:

```typescript
contentfulService.getReleases()       // ‚Üí ContentfulResponse<ContentfulRelease>
contentfulService.getBandcampAlbums() // ‚Üí ContentfulResponse<ContentfulBandcampAlbum>
contentfulService.getArtPieces()      // ‚Üí ContentfulResponse<ContentfulArt>
contentfulService.getBlahgPosts()     // ‚Üí ContentfulResponse<ContentfulBlahg>
```

All methods use include depth 2 (`include: 2`) to resolve one level of nested references.

**`src/utils/contentful.ts`** ‚Äî creates and exports the single Contentful client:

```typescript
const client = createClient({
  space: import.meta.env.VITE_CONTENTFUL_SPACE,
  accessToken: import.meta.env.VITE_CONTENTFUL_ACCESS_TOKEN,
});
```

## Error Handling

All Contentful errors are normalized in `src/utils/error-handler.ts`:

- **`ContentfulError`** ‚Äî extends `Error`; carries `context` (string label) and optional `originalError`
- **`handleContentfulError(error, context)`** ‚Äî always throws a `ContentfulError`; called in service method catch blocks
- **`logError(error, context)`** ‚Äî `console.error` wrapper; only meaningful in dev (Terser drops `console.*` in prod builds)

Pattern used in every service method:
```typescript
try {
  return await client.getEntries(...)
} catch (error) {
  logError(error, 'context label')
  handleContentfulError(error, 'context label') // throws
}
```

## Data Transformation

`src/utils/transformers.ts` converts raw Contentful entries to UI models:

| Function | Purpose |
|----------|---------|
| `createSlug(name)` | Canonical slug: lowercase, hyphens, alphanumeric only |
| `createAlbumMap(albums[])` | Indexes bandcamp albums by Contentful entry ID |
| `transformReleaseData(release, albumMap)` | Raw release ‚Üí `IRelease` UI model |
| `transformTrack(track, albumMap)` | Raw track ‚Üí `ISong` UI model, resolves album ref |
| `findReleaseBySlug(releases[], slug)` | Finds release by slug (createSlug match, case-insensitive fallback) |
| `findTrackBySlug(tracks[], slug)` | Finds track by slug |
| `parseLocalDate(dateString)` | Parses ISO / YYYY-MM-DD strings to `Date` |
